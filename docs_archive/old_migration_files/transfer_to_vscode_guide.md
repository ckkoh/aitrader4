**Then run this helper script:**

```python
# save as: extract_artifacts.py
import re
import os

def extract_artifacts_from_chat(chat_file):
    """
    Extract code artifacts from exported chat
    """
    with open(chat_file, 'r', encoding='utf-8') as f:
        content = f.read()
    
    # Pattern to find code blocks
    # Adjust based on export format
    pattern = r'```python\n(.*?)```'
    
    artifacts = re.findall(pattern, content, re.DOTALL)
    
    print(f"Found {len(artifacts)} code artifacts")
    
    # You'll need to manually match to filenames
    return artifacts

# Usage:
# artifacts = extract_artifacts_from_chat('chat_export.txt')
```

## ü§ñ STEP 4: Set Up Claude Code (Desktop)

### **What is Claude Code?**
Claude Code is a CLI tool that lets Claude help you with coding directly in your terminal.

### **Installation**

```bash
# Install via npm
npm install -g @anthropic-ai/claude-code

# Or via pip (if available)
pip install claude-code
```

### **Configuration**

**1. Set API Key**

```bash
# Add to ~/.bashrc or ~/.zshrc
export ANTHROPIC_API_KEY='your-api-key-here'

# Or create config file
mkdir -p ~/.config/claude-code
echo '{"api_key": "your-api-key-here"}' > ~/.config/claude-code/config.json
```

**2. Configure for Project**

**File: `.claude-code.json`** (in project root)

```json
{
  "project_name": "Trading System",
  "context_files": [
    "README.md",
    "docs/Complete_System_Integration_Guide.md",
    "core/backtesting_engine.py",
    "core/feature_engineering.py"
  ],
  "ignore_patterns": [
    "*.pyc",
    "__pycache__/",
    "venv/",
    "data/",
    "*.db"
  ],
  "ai_preferences": {
    "style": "professional",
    "include_comments": true,
    "include_docstrings": true,
```

## üíª STEP 5: Using Claude Code Effectively

### **Common Commands**

**1. Code Review**
```bash
# Review a file
claude-code review core/backtesting_engine.py

# Review recent changes
claude-code review --diff
```

**2. Generate Code**
```bash
# Generate new feature
claude-code generate "Add email alerts to RecoveryStrategy class"

# Generate tests
claude-code test core/backtesting_engine.py
```

**3. Refactoring**
```bash
# Refactor code
claude-code refactor "Extract position sizing logic into separate class"

# Optimize performance
claude-code optimize core/feature_engineering.py
```

**4. Documentation**
```bash
# Generate docstrings
claude-code document core/ml_training_pipeline.py

# Generate README
claude-code docs --readme
```

**5. Interactive Chat**
```bash
# Start interactive session
claude-code chat

# Then ask questions like:
> How do I add a new technical indicator?
> Explain the walk-forward analysis implementation
> Help me debug this error in backtesting_engine.py
```

### **Project-Specific Workflows**

**Workflow 1: Add New Strategy**
```bash
# 1. Generate strategy template
claude-code generate "Create a new RSI-based mean reversion strategy"

# 2. Review the code
claude-code review strategies/new_strategy.py

# 3. Generate tests
claude-code test strategies/new_strategy.py

# 4. Run backtest
python tools/run_examples.py --custom strategies/new_strategy.py
```

**Workflow 2: Debug Issue**
```bash
# 1. Describe the issue
claude-code debug "Backtest failing with KeyError on 'close' column"

# 2. Get suggestions
claude-code suggest "How to handle missing data in backtesting_engine.py"

# 3. Apply fix
claude-code fix core/backtesting_engine.py --issue "handle missing columns"
```

**Workflow 3: Add Feature**
```bash
# 1. Plan the feature
claude-code plan "Add email notifications to health monitoring system"
```

### **Issue: Can't find modules**
```bash
# Fix Python path
export PYTHONPATH="${PYTHONPATH}:${HOME}/trading_system"

# Or add to files
import sys
sys.path.append('/path/to/trading_system')
```

```bash
# Update imports in all files
```
